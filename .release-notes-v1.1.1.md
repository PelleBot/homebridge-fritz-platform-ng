## LightSensor variant for energy tiles

Neuer **accType `energy-light`** als Alternative zu `energy-temperature`. Beide nutzen dieselbe OBIS-Datenflow-Logik (Fritz!Smart Energy 250 sub-AINs), aber `energy-light` rendert die Werte als **LightSensor** (Einheit `lx`) statt **TemperatureSensor** (Einheit `°C`).

### Hintergrund

Apple Home rendert beide Sensor-Typen unterschiedlich:

- **TemperatureSensor** (`energy-temperature`): prominentes Tile im Raum-View, `°C`-Einheit
- **LightSensor** (`energy-light`): "passiver Sensor" in der Status-Zeile oben, `lx`-Einheit — per Long-Press → "Im Zuhause anzeigen" als Favorit ins Home-Tab holbar

Beide Einheiten (°C, lx) sind semantisch falsch für Energie-Werte. Manche Nutzer empfinden `lx` als abstrakter und weniger irreführend als `°C` (das auf Temperaturmessung schließen lässt).

### Highlights

- 🆕 Neuer `accType: energy-light` parallel zu `energy-temperature` (beide bleiben verfügbar)
- 🧙 Setup-Wizard Schritt 3 (Smart Energy 250) bekommt **Display-Typ-Auswahl** (Radio: Temperatur / Lichtsensor)
- 🔄 Auto-Migration: wenn man via Wizard von Temperatur auf Lichtsensor wechselt, entfernt das Plugin den alten TemperatureSensor-Service sauber

### Breaking Changes

Keine. Bestehende Configs mit `energy-temperature` laufen unverändert weiter.

### Migration für bestehende Nutzer

Pro Smart-Home-Eintrag den `accType` von `"energy-temperature"` auf `"energy-light"` ändern, Homebridge neu starten:

```json
{
  "active": true,
  "name": "Aktueller Verbrauch",
  "ain": "...-1",
  "accType": "energy-light",
  "obisChannel": "current_power"
}
```

Oder den Setup-Wizard noch einmal durchlaufen und in Schritt 3 die LightSensor-Option wählen.

### Installation

```bash
sudo npm install -g @pellebot/homebridge-fritz-platform-hb2@1.1.1
sudo hb-service restart
```

### Bekannte Limitationen

- **HAP minValue für AmbientLightLevel:** 0.0001 (nicht 0 möglich). Bei aktuellem Wert = 0 W zeigt das Tile `0.0001 lx`. Funktional kein Unterschied, kosmetisch leicht verwirrend.
- **LightSensor in Apple Home Status-Bereich:** das Tile erscheint nicht direkt im Raum sondern im Status-Bereich oben. Per Long-Press als Favorit ins Home-Tab holen (siehe README).
