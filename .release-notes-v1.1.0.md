## Setup-Wizard Release

Beim Klick auf das Plugin-Settings-Icon in Config UI X startet ab dieser Version ein geführter 5-Schritt-Wizard — kein Raten mehr bei Feldern wie `accType`, `obisChannel` oder AIN-Format.

### Highlights

- 🧙 **5-Schritt-Wizard:** FritzBox-Verbindung → Geräte-Auswahl → Smart Energy 250 → Router-Switches → Speichern
- 🔍 **Auto-Discovery** aller DECT-Geräte mit `accType`-Vorschlag pro Gerät anhand der Geräte-Eigenschaften
- ⚡ **Smart Energy 250 OBIS-Reader Auto-Setup:** erkennt Sub-AIN-Paare (-1/-2) und legt 4 HomeKit-Tiles auto-an (Aktueller/Gesamt Verbrauch + Aktuelle/Gesamt Einspeisung)
- 🔘 **Router-Switches Section:** Gast-WLAN, WPS, WLAN 2.4/5 GHz, DECT-Basis, AB, Rufumleitung, Reconnect — alles als Checkboxen
- 💾 **Save + Apply:** Zusammenfassung mit JSON-Vorschau (Passwort maskiert), direkter Homebridge-Restart-Button
- ✏️ **Auto-Pre-fill** aus bestehender Config im Verbindungs-Schritt und bei Router-Switches

### Bug-Fixes

- AIN-Suffix-Detection korrekt priorisiert: DECT 500 Lampen (`-1` als HAN-FUN-Sub-Unit) werden nicht mehr fälschlich als OBIS-Reader interpretiert
- Korrekte `@seydx/fritzbox`-Feldnamen verwendet: `.thermostat` (nicht `.hkr`), `.light` (nicht `.colorcontrol`/`.levelcontrol`), `.alert` (boolean)

### Installation

```bash
sudo npm install -g @pellebot/homebridge-fritz-platform-hb2@1.1.0
sudo hb-service restart
```

Bei npm-Cache-Permission-Fehler vorab:
```bash
sudo chown -R 501:20 ~/.npm
```

### Breaking Changes

Keine. Bestehende Configs werden unverändert übernommen. Das alte JSON-Schema-Form ist über den "Save"-Button (bottom-right) im Plugin-Settings-Modal weiterhin erreichbar als Fallback.

### Bekannte Limitationen

- Standalone Config UI X: `homebridge.savePluginConfig()` kann in Standalone-Modus die WebSocket-ACK nicht empfangen → Save mit 3-Sek-Timeout gerennt. Datei wird trotzdem geschrieben.
- Custom-UI-Icon: erscheint im Config UI X Plugin-Tile erst nach Homebridge-Verified-Plugin-Approval (frühestens 13. Juni 2026, siehe Roadmap).
